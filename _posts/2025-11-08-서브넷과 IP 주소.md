# 서브넷과 IP 주소

- *서브넷(Subnet)*과 **IP 주소(IP Address)**는 네트워크에서 서로 관련된 개념이지만, 각각 다른 역할을 수행한다.
- 간단히 말해, **IP 주소**는 네트워크 상에서 장치를 식별하는 **고유한 주소**
- **서브넷**은 큰 네트워크를 더 작은 네트워크로 나누는 **구분 방법**이다.

---

# **IP 주소**

- 인터넷이나 네트워크 상에서 특정 장치를 식별하는 주소이다.
- IP 주소는 두 가지 버전이 있는데, **IPv4**(예: 192.168.0.1)와 **IPv6**(예: 2001:0db8:85a3::8a2e:0370:7334)가 있다.
- 각 기기는 고유한 IP 주소를 할당받아 네트워크에서 통신할 수 있다.

---

# **서브넷**

- IP 주소의 범위를 나누어 네트워크를 더 작은 네트워크 단위로 분리하는 것이다.
- 즉, IP 주소를 효율적으로 사용하고 트래픽을 분산시키기 위해 네트워크를 서브넷으로 나눈다.
- 서브넷 마스크(subnet mask)라는 개념을 통해 IP 주소를 네트워크 부분과 호스트 부분으로 구분한다.
- 예를 들어, 192.168.1.0/24와 같은 형식에서 `/24`는 서브넷 마스크를 나타내며, 네트워크와 호스트의 경계를 지정한다.

---

# **IP 주소와 서브넷의 관계**

- **IP 주소**는 개별 장치에 할당된 **고유 식별자**로, 네트워크에서 장치를 구분하는 역할을 한다.
- 즉, IP 주소는 네트워크 상에서 데이터가 전달될 목적지 또는 출발지를 나타낸다.
- **서브넷**은 **네트워크를 세분화**하여, IP 주소를 보다 효율적으로 관리하고 네트워크의 성능을 최적화하는 역할을 한다.
- 하나의 IP 주소 범위를 여러 서브넷으로 나누어 관리하면, 트래픽을 분리하고 보안을 강화할 수 있다.

---

# **비교**

| **특징** | **IP 주소** | **서브넷** |
| --- | --- | --- |
| **역할** | 네트워크 상의 장치 식별 | 큰 네트워크를 더 작은 네트워크로 분리 |
| **구성 요소** | 네트워크 부분과 호스트 부분 | 네트워크 부분과 서브넷 마스크로 구분 |
| **표현 방식** | IPv4: 192.168.1.10, IPv6: 2001:... | 서브넷 마스크: 255.255.255.0 (또는 /24) |
| **주 사용 목적** | 장치 간 통신을 위한 주소 제공 | 네트워크 성능 향상 및 효율적 관리 |
| **적용 범위** | 개별 장치나 네트워크의 주소 | 네트워크 내에서 IP 주소 관리 및 분할 |
