# TCP 3-way 핸드쉐이크

- *3-Way 핸드셰이크(3-Way Handshake)*는 **TCP(Transmission Control Protocol)**에서 **신뢰성 있는 연결**을 설정하기 위해 사용하는 **연결 과정**이다.
- TCP는 연결 지향 프로토콜로, 클라이언트와 서버 간에 데이터를 전송하기 전에 **세션**을 설정해야 한다.
- 이 과정에서 3-Way 핸드셰이크는 양쪽이 서로 연결을 확인하고, 데이터 전송이 가능한 상태임을 보장한다.

---

# 3-Way 핸드셰이크의 단계

## **SYN (Synchronize)**

- **클라이언트**가 서버에 연결을 요청한다.
- 클라이언트는 자신이 임의로 선택한 **초기 시퀀스 번호(Initial Sequence Number, ISN)**를 포함한 **SYN 패킷**을 서버로 전송한다.
- 이 단계에서 클라이언트는 "연결을 시작할 준비가 되었다"는 신호를 서버에 보낸다.
- SYN 플래그가 설정된 패킷이 전송됨.

```
클라이언트 -> 서버: SYN, ISN(client)
```

## **SYN-ACK (Synchronize-Acknowledge)**

- **서버**는 클라이언트의 SYN 패킷을 받고, 자신의 초기 시퀀스 번호를 포함한 **SYN-ACK 패킷**을 클라이언트에 응답으로 보낸다.
- 서버는 클라이언트의 요청을 수락하며, 동시에 "연결 준비가 완료되었다"는 신호로 자신의 SYN 패킷을 전송한다.
- 이 과정에서 클라이언트의 ISN에 1을 더해 **ACK(승인)** 응답을 포함한다.

```
서버 -> 클라이언트: SYN, ISN(server), ACK(ISN(client)+1)
```

## **ACK (Acknowledge)**

- **클라이언트**는 서버로부터 받은 SYN-ACK 패킷을 확인하고, 서버의 시퀀스 번호에 1을 더한 **ACK 패킷**을 서버에 전송한다.
- 이로써 서버와 클라이언트가 서로의 준비 상태를 확인하며, 연결이 확립된다.

```
클라이언트 -> 서버: ACK(ISN(server)+1)
```

- 이 세 단계가 완료되면, 클라이언트와 서버 간에 **TCP 연결**이 성공적으로 설정된다.
- 이후 양측은 데이터를 전송할 수 있는 상태가 된다.

---

# 3-Way 핸드셰이크의 목적

## **양측의 준비 상태 확인**:

- 클라이언트와 서버가 서로 통신할 준비가 되었음을 확인한다.

## **초기 시퀀스 번호 동기화**:

- 서로 다른 시퀀스 번호를 동기화하여 데이터 전송 시 순서를 관리할 수 있게 한다.

## **신뢰성 보장**:

- 데이터가 손실되지 않도록 안정적인 연결을 확립한다.

---

# 요약

- **1단계(SYN)**: 클라이언트가 서버에 연결 요청.
- **2단계(SYN-ACK)**: 서버가 연결 요청을 승인하고 응답.
- **3단계(ACK)**: 클라이언트가 서버의 응답을 확인하며 연결 완료.
- 이 3-Way 핸드셰이크 과정을 통해 TCP는 **신뢰성 있는 양방향 통신**을 위한 연결을 설정한다.
