# 이벤트(Event) 처리 매커니즘

- 스프링에서 **이벤트(Event) 처리 매커니즘**은 느슨한 결합(Loose Coupling을 지원하는 중요한 기능으로, 애플리케이션 내에서 발생하는 이벤트를 특정 컴포넌트가 처리할 수 있도록 한다.
- **이벤트는 느슨한 결합**을 유지하면서 애플리케이션 내에서 **비동기 및 동기적 작업**을 처리하는 데 매우 유용하다.
- 이벤트 기반 설계를 통해 컴포넌트 간의 의존성을 줄이고 더 유연한 구조를 만들 수 있다.
- 이벤트 기반 아키텍처를 사용하면, 애플리케이션의 컴포넌트 간에 강한 의존성을 줄이고 더 유연한 설계가 가능해진다.
- 스프링에서 이벤트 처리 매커니즘은 크게 **이벤트 발행자(Publisher)**, **이벤트 리스너(Listener)**, 그리고 이벤트 객체(Event Object로 구성된다.
- 스프링의 이벤트 처리 매커니즘은 주로 **비동기** 및 **동기식** 이벤트를 처리하는 데 사용된다.

# **이벤트 처리 구성 요소**

- 스프링의 이벤트 처리 시스템은 다음의 세 가지 주요 구성 요소로 이루어진다.
- **이벤트 객체(Event Object)**:
    - 이벤트를 표현하는 객체이다. 이 객체는 애플리케이션 내에서 발생하는 특정 사건을 나타낸다.
- **이벤트 발행자(Event Publisher)**:
    - 이벤트가 발생했을 때 이를 발행(전송)하는 역할을 한다.
- **이벤트 리스너(Event Listener)**:
    - 발행된 이벤트를 수신하고, 해당 이벤트에 대한 처리 로직을 수행하는 컴포넌트이다.

# 구현

## **이벤트 객체 (Event Object)**

- 스프링의 이벤트 객체는 **`ApplicationEvent`** 클래스를 상속하여 정의할 수 있다.
- 이는 스프링에서 발생한 이벤트를 나타내는 기본 클래스이다.

### **예시: 이벤트 객체 정의**

```java
import org.springframework.context.ApplicationEvent;

public class UserRegistrationEvent extends ApplicationEvent {

    private String userEmail;

    public UserRegistrationEvent(Object source, String userEmail) {
        super(source);
        this.userEmail = userEmail;
    }

    public String getUserEmail() {
        return userEmail;
    }
}
```

- **`ApplicationEvent`**: 모든 이벤트 클래스는 이 클래스를 상속해야 한다.
- **`source`**: 이벤트가 발생한 객체(발행자)를 나타낸다.
- **`userEmail`**: 사용자 등록 이벤트와 관련된 추가 정보(이메일)를 담고 있다.

## **이벤트 발행자 (Event Publisher)**

- 스프링에서 이벤트 발행자는 **`ApplicationEventPublisher`** 인터페이스를 사용하여 이벤트를 발행할 수 있다.
- 스프링 컨텍스트는 이벤트를 처리하기 위한 발행 시스템을 자동으로 관리한다.

### **예시: 이벤트 발행자 구현**

```java
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.ApplicationEventPublisher;
import org.springframework.stereotype.Service;

@Service
public class UserService {

    @Autowired
    private ApplicationEventPublisher eventPublisher;

    public void registerUser(String userEmail) {
        // 사용자 등록 로직 수행
        System.out.println("사용자가 등록되었습니다: " + userEmail);

        // 사용자 등록 이벤트 발행
        UserRegistrationEvent event = new UserRegistrationEvent(this, userEmail);
        eventPublisher.publishEvent(event);
    }
}
```

- **`ApplicationEventPublisher`**: 이벤트 발행을 담당하는 스프링 인터페이스이다.
- `publishEvent()`: 이벤트가 발생하면 이 메서드를 호출하여 이벤트를 발행한다.

## **이벤트 리스너 (Event Listener)**

- 이벤트 리스너는 이벤트가 발행되면 이를 수신하여 필요한 처리를 하는 역할을 한다.
- 스프링에서는 **`@EventListener`** 애너테이션을 사용하여 이벤트 리스너를 구현할 수 있다.

### **예시: 이벤트 리스너 구현**

```java
import org.springframework.context.event.EventListener;
import org.springframework.stereotype.Component;

@Component
public class UserRegistrationEventListener {

    @EventListener
    public void handleUserRegistrationEvent(UserRegistrationEvent event) {
        System.out.println("이벤트 수신: 사용자가 등록되었습니다: " + event.getUserEmail());
        // 추가적인 처리 로직
    }
}
```

- **`@EventListener`**: 이 애너테이션을 사용하여 해당 메서드가 특정 이벤트를 수신하도록 설정한다.
- 이벤트 리스너는 발행된 이벤트를 자동으로 수신하고, 해당 이벤트 객체를 처리할 수 있다.

## **비동기 이벤트 처리**

- 스프링에서 이벤트 리스너는 기본적으로 **동기식**으로 동작한다.
- 하지만 비동기 처리가 필요한 경우 **`@Async`** 애너테이션을 추가하여 비동기 이벤트 처리를 구현할 수 있다.

### **비동기 이벤트 리스너 예시**

```java
import org.springframework.context.event.EventListener;
import org.springframework.scheduling.annotation.Async;
import org.springframework.stereotype.Component;

@Component
public class AsyncUserRegistrationEventListener {

    @Async  // 비동기적으로 이벤트 처리
    @EventListener
    public void handleUserRegistrationEvent(UserRegistrationEvent event) {
        System.out.println("비동기 이벤트 수신: 사용자가 등록되었습니다: " + event.getUserEmail());
        // 추가적인 처리 로직
    }
}
```

- **`@Async`**: 이 애너테이션을 추가하면 해당 이벤트 리스너는 비동기적으로 동작하게 된다. 비동기 처리를 위해서는 `@EnableAsync` 애너테이션을 사용하여 스프링에서 비동기 기능을 활성화해야 한다.

```java
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.EnableAsync;

@Configuration
@EnableAsync
public class AsyncConfig {
    // 비동기 처리를 위한 설정
}
```

## **이벤트 필터링 및 커스텀 이벤트 리스너 조건**

- 스프링은 이벤트 필터링을 위한 다양한 방법을 제공한다.
- 예를 들어, 특정 조건에 맞는 이벤트만 처리하고 싶을 때는 `@EventListener`의 `condition` 속성을 사용할 수 있다.

### **조건부 이벤트 리스너**

```java
@Component
public class ConditionalUserRegistrationListener {

    @EventListener(condition = "#event.userEmail.endsWith('.com')")
    public void handleUserRegistrationEvent(UserRegistrationEvent event) {
        System.out.println(".com 도메인을 가진 사용자가 등록되었습니다: " + event.getUserEmail());
    }
}
```

- **`condition` 속성**: SpEL(Spring Expression Language)을 사용하여 조건을 정의할 수 있다. 조건을 만족하는 이벤트만 리스너가 처리한다.

## **글로벌 이벤트 리스너**

- `ApplicationListener` 인터페이스를 직접 구현하여 모든 이벤트를 처리하는 **글로벌 이벤트 리스너**를 만들 수 있다.

### **글로벌 이벤트 리스너 구현**

```java
import org.springframework.context.ApplicationListener;
import org.springframework.stereotype.Component;

@Component
public class GlobalEventListener implements ApplicationListener<ApplicationEvent> {

    @Override
    public void onApplicationEvent(ApplicationEvent event) {
        System.out.println("글로벌 이벤트 수신: " + event.getClass().getSimpleName());
    }
}
```

- **`ApplicationListener<T extends ApplicationEvent>`**: 특정 이벤트 유형에 대해 리스너를 작성할 수 있으며, 이 경우 모든 `ApplicationEvent` 하위 이벤트를 수신할 수 있다.

### **이벤트 계층 구조**

- 스프링 이벤트는 계층 구조를 가질 수 있다.
- 예를 들어, 하나의 이벤트 클래스가 다른 이벤트 클래스를 상속하면, 해당 이벤트 리스너는 하위 클래스의 이벤트도 처리할 수 있다.

### **이벤트 상속 예시**

```java
public class UserEvent extends ApplicationEvent {
    // 공통 로직
}

public class UserRegistrationEvent extends UserEvent {
    // 사용자 등록 로직
}

@Component
public class UserEventListener {

    @EventListener
    public void handleUserEvent(UserEvent event) {
        System.out.println("사용자 이벤트 수신");
    }
}
```

- 이벤트 리스너는 상위 클래스의 이벤트를 처리할 수 있으며, 하위 클래스 이벤트도 처리 가능하다.

## **스프링 내장 이벤트**

- 스프링은 자체적으로 다양한 내장 이벤트를 제공한다.
- 이 이벤트들은 스프링 컨텍스트의 시작, 종료, 리프레시 등과 같은 중요한 라이프사이클 이벤트에 대한 정보를 제공한다.
- **`ContextRefreshedEvent`**: 애플리케이션 컨텍스트가 초기화되거나 리프레시될 때 발생.
- **`ContextClosedEvent`**: 애플리케이션 컨텍스트가 종료될 때 발생.
- **`ContextStartedEvent`**: 애플리케이션 컨텍스트가 시작될 때 발생.
- **`ContextStoppedEvent`**: 애플리케이션 컨텍스트가 중지될 때 발생.

### **내장 이벤트 리스너 구현**

```java
import org.springframework.context.event.ContextRefreshedEvent;
import org.springframework.context.event.EventListener;
import org.springframework.stereotype.Component;

@Component
public class ApplicationContextEventListener {

    @EventListener
    public void handleContextRefresh(ContextRefreshedEvent event) {
        System.out.println("애플리케이션 컨텍스트가 리프레시되었습니다.");
    }
}
```
