# CDN이란

- CDN(Content Delivery Network)은 **콘텐츠 전송 네트워크**로, **사용자와 가까운 서버에 콘텐츠를 캐싱**하여 **웹 콘텐츠**(이미지, 비디오, 스타일 시트, 자바스크립트 파일 등)를 빠르게 전송하는 **분산된 서버 네트워크**이다.
- CDN의 주된 목적은 **웹 페이지 로딩 속도를 향상**시키고, **서버 부하를 줄이며**, 네트워크 장애 발생 시에도 **안정성을 높이는 것**이다.

---

# CDN의 주요 기능

## **지리적 분산**:

- CDN은 전 세계 여러 곳에 위치한 서버들로 구성되어 있다.
- 사용자가 콘텐츠를 요청할 때, 지리적으로 가장 가까운 서버에서 콘텐츠를 제공하여 전송 속도를 극대화한다.

## **캐싱**:

- CDN은 원본 서버(오리진 서버)에서 한 번 가져온 콘텐츠를 캐싱하여, 이후의 요청에 대해 해당 콘텐츠를 제공할 때 원본 서버에 재요청하지 않고 빠르게 응답한다.

## **부하 분산**:

- 여러 서버에 트래픽을 분산시켜 오리진 서버의 부하를 줄이고, 과부하나 트래픽 폭증에 대비할 수 있다.

## **보안 강화**:

- CDN은 DDoS 공격 방어, SSL 암호화 등 다양한 보안 기능을 제공하여, 악의적인 트래픽을 차단하고 네트워크를 보호할 수 있다.

---

# CDN의 작동 방식

## **콘텐츠 요청**:

- 사용자가 웹사이트에 접속하면, 브라우저는 해당 콘텐츠(이미지, 비디오, CSS 파일 등)를 요청한다.
- CDN이 설정된 경우, 브라우저는 **CDN 서버**에서 캐싱된 콘텐츠를 가져오려고 시도한다.

## **가장 가까운 CDN 서버 선택**:

- CDN은 사용자의 지리적 위치를 기준으로 가장 가까운 서버(에지 서버, Edge Server)로 요청을 보낸다. 예를 들어, 사용자가 한국에 있다면 한국에 위치한 CDN 서버에서 콘텐츠를 제공한다.

## **콘텐츠 제공**:

- **캐시된 콘텐츠**가 CDN 서버에 있는 경우, 해당 서버는 즉시 콘텐츠를 사용자에게 전송한다.
- **캐시된 콘텐츠가 없는 경우**, CDN 서버는 원본 서버(오리진 서버)에 요청하여 콘텐츠를 가져온 뒤, 이를 캐싱하고 사용자에게 전달한다.

## **캐시 만료 및 갱신**:

- 캐시된 콘텐츠는 일정 기간 동안 유지되며, 캐시 만료 기간이 지나면 CDN 서버는 오리진 서버로부터 최신 콘텐츠를 다시 받아 캐싱한다.

---

# CDN의 이점

## **콘텐츠 전송 속도 향상**:

- 사용자가 지리적으로 가까운 서버에서 콘텐츠를 받기 때문에 웹 페이지 로딩 시간이 단축된다.
- 특히, 글로벌 웹사이트에서 효과적이다.

## **서버 부하 감소**:

- CDN이 콘텐츠를 캐싱하여 분산시킴으로써, 원본 서버로의 요청이 줄어들고 서버 부하가 크게 감소한다.

## **트래픽 관리**:

- CDN은 여러 서버에 트래픽을 분산시키기 때문에 대규모 트래픽이나 갑작스러운 트래픽 증가에도 안정적으로 대응할 수 있다.

## **고가용성 및 안정성**:

- CDN은 전 세계 여러 서버에 콘텐츠를 분산 저장하기 때문에, 특정 서버에 문제가 생겨도 다른 서버에서 콘텐츠를 제공할 수 있어 안정성이 높아진다.

## **보안 강화**:

- CDN은 DDoS 공격 방어, SSL 인증서 관리, 웹 애플리케이션 방화벽(WAF) 등을 통해 보안을 강화할 수 있다.

---

# CDN의 사용 사례

## **글로벌 웹사이트**:

- 글로벌 사용자에게 빠르게 웹사이트 콘텐츠를 제공하기 위해 CDN이 사용된다.

## **스트리밍 서비스**:

- 실시간 비디오 스트리밍이나 대용량 미디어 콘텐츠 제공을 최적화하기 위해 CDN이 필요하다.

## **전자상거래**:

- 쇼핑몰 사이트에서 이미지, 제품 데이터 등의 빠른 로딩을 위해 CDN이 사용된다.

## **모바일 애플리케이션**:

- 모바일 앱의 데이터를 효율적으로 제공하기 위해 CDN이 적용된다.

---

# 결론

- CDN은 사용자와 가까운 서버에서 콘텐츠를 제공하여 웹사이트의 성능을 크게 향상시키는 핵심 기술이다.
- 이를 통해 웹 페이지 로딩 속도를 빠르게 하고, 서버 부하를 줄이며, 네트워크 트래픽을 효율적으로 분산시킬 수 있다.
- 특히, 대규모 사용자 트래픽을 처리해야 하는 글로벌 서비스에서 필수적인 요소로 자리잡고 있다.
