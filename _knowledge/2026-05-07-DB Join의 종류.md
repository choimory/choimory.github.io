# DB Join

---

# **INNER JOIN**

- 두 테이블 간에 **공통된 값이 있는 행**만 결합하여 반환한다. 즉, 두 테이블 모두에서 일치하는 데이터만 결과에 포함된다.
- 일치하지 않는 행은 결과에 포함되지 않는다.
- 가장 많이 사용되는 JOIN 유형.
- **예시**: A 테이블과 B 테이블에서 공통된 키를 기준으로 두 테이블에서 일치하는 데이터만 반환.
    
    ```sql
    SELECT A.*, B.*
    FROM A
    INNER JOIN B ON A.id = B.id;
    ```
    
    - **결과**: A 테이블과 B 테이블의 `id` 값이 일치하는 행만 반환된다.

---

# **LEFT OUTER JOIN**

- 왼쪽 테이블(A)의 모든 행을 반환하고, 오른쪽 테이블(B)의 일치하는 데이터가 있으면 함께 반환한다. 오른쪽 테이블에 일치하는 데이터가 없으면 **NULL**로 채운다.
- 왼쪽 테이블의 데이터는 모두 포함되며, 오른쪽 테이블에 대응하는 데이터가 없으면 NULL 값이 반환된다.
- **예시**: A 테이블의 모든 데이터와 B 테이블에서 일치하는 데이터, 없으면 NULL.
    
    ```sql
    SELECT A.*, B.*
    FROM A
    LEFT JOIN B ON A.id = B.id;
    ```
    
    - **결과**: A 테이블의 모든 행을 반환하고, B 테이블에 일치하지 않는 경우에는 NULL로 반환된다.

---

# **RIGHT OUTER JOIN**

- 오른쪽 테이블(B)의 모든 행을 반환하고, 왼쪽 테이블(A)의 일치하는 데이터가 있으면 함께 반환한다. 왼쪽 테이블에 일치하는 데이터가 없으면 **NULL**로 채운다.
- 오른쪽 테이블의 데이터는 모두 포함되며, 왼쪽 테이블에 대응하는 데이터가 없으면 NULL 값이 반환된다.
- **예시**: B 테이블의 모든 데이터와 A 테이블에서 일치하는 데이터, 없으면 NULL.
    
    ```sql
    SELECT A.*, B.*
    FROM A
    RIGHT JOIN B ON A.id = B.id;
    ```
    
    - **결과**: B 테이블의 모든 행을 반환하고, A 테이블에 일치하지 않는 경우에는 NULL로 반환된다.

---

# **FULL OUTER JOIN**

- 두 테이블 모두에서 일치하는 행을 반환하고, 일치하지 않는 행도 각각 반환된다. 즉, **LEFT JOIN**과 **RIGHT JOIN**의 결합 결과라고 볼 수 있다. 어느 한쪽 테이블에 일치하는 데이터가 없으면 **NULL**로 반환된다.
- 두 테이블의 모든 행을 포함하여 결과를 반환한다.
- 한쪽 테이블에 대응하는 값이 없으면 NULL 값이 반환된다.
- **예시**: A 테이블과 B 테이블에서 일치하는 데이터뿐만 아니라, 양쪽에 일치하지 않는 데이터도 모두 반환.
    
    ```sql
    SELECT A.*, B.*
    FROM A
    FULL JOIN B ON A.id = B.id;
    ```
    
    - **결과**: A 테이블과 B 테이블의 모든 행을 반환하고, 일치하지 않는 부분은 NULL로 채운다.

---

# **CROSS JOIN**

- 두 테이블 간에 **카티전 곱(Cartesian Product)**을 수행하여, 왼쪽 테이블의 각 행과 오른쪽 테이블의 각 행을 모두 결합한 결과를 반환한다. 테이블 간의 모든 가능한 조합이 만들어진다.
- 두 테이블에 각각 n개, m개의 행이 있을 경우, n * m개의 결과를 반환한다.
- 실무에서는 자주 사용되지 않으며, 특정 계산이나 조합을 만들 때 유용하다.
- **예시**: A 테이블의 모든 행과 B 테이블의 모든 행의 조합.
    
    ```sql
    SELECT A.*, B.*
    FROM A
    CROSS JOIN B;
    ```
    
    - **결과**: A 테이블의 모든 행과 B 테이블의 모든 행의 조합을 반환한다.

---

# **SELF JOIN**

- 동일한 테이블 내에서 두 개의 인스턴스를 서로 결합하는 JOIN이다. 주로 트리 구조나 계층적 데이터를 처리할 때 사용된다.
- 하나의 테이블을 두 번 참조하여, 같은 테이블 내에서 관계를 정의하는 경우 사용된다.
- **예시**: A 테이블에서 자기 자신과 결합하여 관련 데이터를 조회.
    
    ```sql
    SELECT A.*, B.*
    FROM A AS A1
    INNER JOIN A AS A2 ON A1.id = A2.parent_id;
    ```
    
    - **결과**: A 테이블에서 자기 자신과의 관계를 통해 데이터 조회.

---

# **NATURAL JOIN**

- 두 테이블 간의 **공통 컬럼**을 기준으로 자동으로 JOIN을 수행하는 방식이다. 공통된 컬럼의 이름이 동일해야 하며, 일치하는 값이 있는 경우만 결합된다.
- 공통된 컬럼을 명시하지 않아도 자동으로 일치하는 컬럼을 찾아 결합한다.
- 컬럼 이름이 동일할 때만 작동하므로, 예상치 못한 결과를 가져올 수 있어 주의가 필요하다.
- **예시**: 공통된 컬럼 이름을 기준으로 결합.
    
    ```sql
    SELECT *
    FROM A
    NATURAL JOIN B;
    ```
    
    - **결과**: A 테이블과 B 테이블 간에 공통된 컬럼을 기준으로 일치하는 행만 결합한다.

# 비교

| JOIN 유형 | 설명 | 특징 |
| --- | --- | --- |
| **INNER JOIN** | 두 테이블에서 공통된 값을 가진 행만 반환 | 일치하지 않는 행은 포함되지 않음 |
| **LEFT JOIN** | 왼쪽 테이블의 모든 행을 반환하고, 오른쪽 테이블과 일치하지 않으면 NULL | 왼쪽 테이블에 모든 데이터 포함 |
| **RIGHT JOIN** | 오른쪽 테이블의 모든 행을 반환하고, 왼쪽 테이블과 일치하지 않으면 NULL | 오른쪽 테이블에 모든 데이터 포함 |
| **FULL JOIN** | 두 테이블 모두에서 일치하지 않는 데이터도 포함하여 반환 | 어느 한쪽에 일치하지 않는 경우 NULL 반환 |
| **CROSS JOIN** | 두 테이블의 모든 행을 조합하여 반환 | 카티전 곱(Cartesian Product) 수행 |
| **SELF JOIN** | 같은 테이블을 두 번 참조하여 결합 | 자기 자신과의 관계를 표현하는 데 사용 |
| **NATURAL JOIN** | 공통된 컬럼 이름을 기준으로 자동으로 결합 | 공통 컬럼이 자동으로 결합되며, 컬럼 이름이 동일해야 함 |

---

# 정리

- **INNER JOIN**: 공통된 데이터만 반환.
- **LEFT/RIGHT JOIN**: 한쪽 테이블의 모든 데이터를 포함하여 결합.
- **FULL JOIN**: 양쪽 테이블의 모든 데이터를 결합.
- **CROSS JOIN**: 두 테이블 간 모든 가능한 조합을 반환.
- **SELF JOIN**: 동일한 테이블을 결합하여 자기 참조 구조를 표현.
- **NATURAL JOIN**: 공통된 컬럼 이름을 기준으로 결합.
