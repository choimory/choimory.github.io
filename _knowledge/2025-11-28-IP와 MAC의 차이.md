# IP와 MAC의 차이

- **IP 주소**와 **MAC 주소**는 모두 네트워크 상에서 장치를 식별하는 데 사용되지만, 각각의 용도와 기능은 다르다.
- IP 주소는 **네트워크 계층**에서 사용되는 **논리적 주소**
- MAC 주소는 **데이터 링크 계층**에서 사용되는 **물리적 주소**

---

# **IP 주소(IP Address)**

- **역할**:
    - IP 주소는 네트워크 상에서 장치 간 데이터를 전송할 때 사용되는 **논리적 주소**이다.
    - 네트워크 계층(OSI 모델의 3계층)에서 사용되며, 네트워크와 호스트를 구분하는 정보를 포함한다.
- **변경 가능성**:
    - IP 주소는 **동적**으로 할당될 수 있다.
    - 예를 들어, 네트워크에 연결될 때마다 새로운 IP 주소를 할당받을 수 있다.
    - DHCP(Dynamic Host Configuration Protocol)를 통해 IP 주소를 동적으로 할당받기도 한다.
- **주소 구조**:
    - **IPv4**: 32비트 주소로, 4개의 8비트로 구성된 숫자로 표현된다. 예: `192.168.1.10`
    - **IPv6**: 128비트 주소로, 더 많은 주소 공간을 제공한다. 예: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`
- **주 사용 목적**:
    - IP 주소는 네트워크 상에서 데이터를 전송할 때 **출발지와 목적지**를 나타내는 역할을 한다.
    - 인터넷에서 장치가 서로를 찾아 데이터를 교환하는 데 사용된다.
- **주 사용 위치**:
    - **네트워크 간** 통신.
    - 로컬 네트워크뿐만 아니라, 장치가 인터넷 등 여러 네트워크와 통신할 때 사용된다.

---

# **MAC 주소(MAC Address)**

- **역할**:
    - MAC 주소는 네트워크 카드 또는 네트워크 인터페이스 컨트롤러(NIC)에 할당된 **고유한 물리적 주소**로, 데이터 링크 계층(OSI 모델의 2계층)에서 사용된다.
    - MAC 주소는 장치가 로컬 네트워크(LAN)에서 통신할 때 사용된다.
- **변경 불가능성**:
    - 일반적으로 **하드웨어에 고정**되어 있으며, 제조업체에서 장치에 할당된 고유한 식별 번호이다.
    - 그러나 특정 상황에서는 소프트웨어를 통해 MAC 주소를 변경할 수 있다.
- **주소 구조**:
    - MAC 주소는 48비트로 구성되며, 16진수로 표현된다.
    - 보통 6개의 2자리 16진수로 표시되며, 각 2자리 사이에 콜론(`:`) 또는 하이픈()이 들어간다. 예: `00:1A:2B:3C:4D:5E`
- **주 사용 목적**:
    - MAC 주소는 **로컬 네트워크에서 데이터 프레임**을 전달할 때 사용된다.
    - 네트워크 스위치가 MAC 주소를 기반으로 네트워크 내 장치들 간의 통신을 관리한다.
- **주 사용 위치**:
    - **같은 네트워크 내**에서 장치들 간의 통신. 주로 LAN 환경에서 데이터를 전달할 때 사용된다.

---

# **IP 주소와 MAC 주소의 상호작용**

- **IP 주소**는 네트워크 계층에서 통신하는 장치의 논리적 위치를 나타내며, 인터넷이나 외부 네트워크와 통신할 때 사용된다.
- **MAC 주소**는 동일한 네트워크 내에서 데이터가 올바른 장치에 도달하도록 보장하는 역할을 하며, 로컬 네트워크에서 데이터를 주고받을 때 사용된다.
- 이 두 주소는 **ARP(Address Resolution Protocol)**를 통해 연계되어 작동한다.
- 예를 들어, 한 장치가 로컬 네트워크 내에서 다른 장치로 데이터를 보내려고 할 때, IP 주소를 MAC 주소로 변환하는 과정을 통해 데이터가 올바른 장치에 도달하게 된다.

---

# **활용 예시**

1. **IP 주소 활용 예시**:
    - 사용자가 웹 브라우저에서 `www.example.com`을 입력하면, DNS 서버는 해당 도메인에 대한 IP 주소를 반환한다.
    - 사용자의 장치는 IP 주소를 통해 웹 서버에 데이터를 요청하고, 웹 서버는 IP 주소를 통해 응답을 보내준다.
2. **MAC 주소 활용 예시**:
    - 로컬 네트워크에서 프린터에 문서를 전송하려고 할 때, 사용자의 장치는 프린터의 IP 주소를 알고 있어도, 실제 데이터 전송은 **MAC 주소**를 통해 이루어진다.
    - ARP를 통해 프린터의 IP 주소에 해당하는 MAC 주소를 찾아, 네트워크 스위치가 이를 기반으로 데이터 프레임을 프린터로 전달한다.

# 비교

| **특징** | **IP 주소** | **MAC 주소** |
| --- | --- | --- |
| **역할** | 네트워크 상에서 장치를 식별하는 논리적 주소 | 네트워크 인터페이스의 고유 물리적 주소 |
| **계층** | 네트워크 계층 (OSI 3계층) | 데이터 링크 계층 (OSI 2계층) |
| **변경 가능성** | 네트워크에 따라 동적으로 변경 가능 | 보통 변경 불가능, 제조 시 고정됨 |
| **표현 방식** | IPv4: `192.168.1.1` / IPv6: `2001:0db8::1` | `00:1A:2B:3C:4D:5E` (16진수) |
| **사용 위치** | 네트워크 간의 통신 (인터넷, 외부 네트워크) | 로컬 네트워크(LAN) 내 통신 |
| **주소 할당 방식** | 네트워크 관리자에 의해 동적/정적 할당 | 제조업체에 의해 하드웨어에 고유하게 할당 |
| **식별 대상** | 네트워크 상의 호스트 및 장치 | 네트워크 카드(NIC) 또는 인터페이스 |

---

# 결론

- **IP 주소**는 장치가 **네트워크 계층**에서 통신할 때 사용하는 **논리적 주소**이고, **MAC 주소**는 장치가 **데이터 링크 계층**에서 로컬 네트워크 내에서 통신할 때 사용하는 **물리적 주소**이다.
- IP 주소는 네트워크 상에서 변경될 수 있지만, MAC 주소는 일반적으로 고정되어 있으며, 두 주소는 ARP를 통해 상호작용하여 원활한 네트워크 통신을 가능하게 한다.
