# 라우팅(Routing)과 스위칭(Switching)의 차이

- *라우팅(Routing)*과 **스위칭(Switching)**은 네트워크 통신에서 중요한 역할을 하는 두 가지 개념이지만, 그들이 처리하는 **데이터 전송 범위**와 **방식**은 다르다.
- 라우팅은 주로 **다른 네트워크 간의 데이터 전송**을 담당하는 반면, 스위칭은 **동일 네트워크 내의 데이터 전송**을 처리한다.

---

# **라우팅(Routing)**

- **라우팅**은 **다른 네트워크 간**의 데이터 전송 경로를 결정하는 과정이다.
- 라우터가 이 역할을 수행하며, **IP 주소**를 기반으로 데이터를 최적의 경로로 전달한다.
- 라우팅은 주로 **네트워크 계층(OSI 7계층 중 3계층)**에서 작동하며, 서로 다른 네트워크 간에 데이터를 이동시키기 위해 필요하다.

---

# 라우팅의 주요 특징:

- **네트워크 간 데이터 전송**:
    - 라우팅은 하나의 네트워크에서 다른 네트워크로 데이터를 전송하는 작업을 수행한다.
    - 예를 들어, 가정 내 네트워크에서 외부 인터넷 네트워크로 데이터를 보내는 과정에 라우팅이 사용된다.
- **IP 주소 기반**:
    - 라우팅은 **논리적 주소**인 **IP 주소**를 사용하여 데이터가 목적지까지 가는 최적의 경로를 결정한다.
    - IP 주소는 네트워크와 호스트를 구분하는데, 라우터는 이 정보를 사용하여 올바른 네트워크로 데이터를 전송한다.
- **라우팅 테이블**:
    - 라우터는 **라우팅 테이블**을 유지하여 어떤 경로로 패킷을 전송할지 결정한다.
    - 이 테이블에는 네트워크의 목적지와 그 목적지로 가기 위한 최적 경로가 기록되어 있다.
- **동적 라우팅과 정적 라우팅**:
    - **정적 라우팅**은 관리자가 수동으로 경로를 설정하는 방식이다.
    - **동적 라우팅**은 라우터가 네트워크 상태에 따라 **라우팅 프로토콜**(예: OSPF, BGP)을 사용해 경로를 자동으로 선택하는 방식이다.

---

# 라우팅의 예시:

- 가정에서 PC가 외부의 웹 서버에 연결될 때, **라우터**는 해당 웹 서버의 네트워크로 데이터를 전달하는 경로를 결정하고, 패킷을 해당 경로로 전송한다.

---

# **스위칭(Switching)**

- **스위칭**은 **동일 네트워크 내**에서 장치들 간에 데이터를 전달하는 역할을 한다.
- 스위치는 **MAC 주소**를 기반으로 데이터를 전달하며, **데이터 링크 계층(OSI 7계층 중 2계층)**에서 작동한다.
- 스위칭은 주로 **LAN(Local Area Network)**에서 사용된다.

---

# 스위칭의 주요 특징:

## **네트워크 내 데이터 전송**:

- 스위칭은 같은 네트워크에 있는 장치들 사이에서 데이터를 전달하는 과정이다.
- 예를 들어, 회사 내의 PC 간 데이터 전송이나 파일 공유는 스위칭을 통해 이루어진다.

## **MAC 주소 기반**:

- 스위칭은 **물리적 주소**인 **MAC 주소**를 사용하여 네트워크 내에서 데이터를 전달한다.
- 스위치는 각 장치의 MAC 주소를 기반으로 어떤 포트로 데이터를 보내야 하는지 결정한다.

## **스위칭 테이블**:

- 스위치는 각 네트워크 장치의 MAC 주소와 해당 장치가 연결된 포트를 저장한 **스위칭 테이블**을 유지한다.
- 이를 통해 네트워크 내에서 데이터를 효율적으로 전달할 수 있다.

## **유니캐스트, 멀티캐스트, 브로드캐스트**:

- **유니캐스트**: 특정 하나의 장치로 데이터가 전송된다.
- **멀티캐스트**: 특정 그룹의 장치에 데이터가 전송된다.
- **브로드캐스트**: 네트워크 내 모든 장치로 데이터가 전송된다.

---

# 스위칭의 예시:

- 회사의 내부 네트워크에서 **스위치**는 각 PC와 연결된 포트를 인식하고, 데이터가 올바른 장치로 전달되도록 한다.
- 예를 들어, PC1에서 PC2로 데이터를 보낼 때 스위치는 PC2의 MAC 주소를 기반으로 데이터를 전달한다.

---

# **비교**

| **특징** | **라우팅(Routing)** | **스위칭(Switching)** |
| --- | --- | --- |
| **동작 계층** | 네트워크 계층(OSI 3계층) | 데이터 링크 계층(OSI 2계층) |
| **사용하는 주소** | IP 주소(논리적 주소) | MAC 주소(물리적 주소) |
| **역할** | 네트워크 간의 데이터 전송 | 동일 네트워크 내 데이터 전송 |
| **장비** | 라우터 | 스위치 |
| **사용 범위** | WAN, 인터넷 등 여러 네트워크 간의 통신 | LAN 같은 동일 네트워크 내의 통신 |
| **라우팅/스위칭 테이블** | 라우팅 테이블을 사용하여 최적 경로 결정 | 스위칭 테이블을 사용하여 포트 결정 |
| **유형** | 동적/정적 라우팅 | 유니캐스트, 멀티캐스트, 브로드캐스트 |

---

# 결론

- **라우팅**은 **서로 다른 네트워크 간**의 통신을 관리하며, 라우터가 **IP 주소**를 기반으로 경로를 결정해 데이터를 전달한다. 인터넷처럼 네트워크가 복잡하게 얽힌 환경에서 사용된다.
- **스위칭**은 **동일 네트워크 내**에서 장치 간 데이터를 전송하며, 스위치는 **MAC 주소**를 사용해 데이터를 올바른 장치로 전달한다. 주로 LAN 환경에서 사용된다.
