# Vector Database 개념

Vector Database는 고차원 벡터 데이터를 저장하고 검색하는 특수 목적 데이터베이스. 전통적인 관계형 DB가 정확한 값 매칭에 최적화된 반면, Vector DB는 유사도 검색(similarity search)에 최적화됨.

## 핵심 개념

### 벡터 임베딩(Vector Embedding)

- 텍스트, 이미지, 오디오 등을 고차원 숫자 배열로 변환한 표현
- 의미적으로 유사한 데이터는 벡터 공간에서 가까이 위치
- 예시: "개"와 "강아지"는 유사한 벡터 값을 가짐
- 일반적으로 128차원에서 1536차원 사이의 배열

### 유사도 측정 방식

- **코사인 유사도(Cosine Similarity)**: 벡터 간 각도 측정, 방향 중심
- **유클리드 거리(Euclidean Distance)**: 벡터 간 직선 거리, 절대적 위치 중심
- **내적(Dot Product)**: 벡터 크기와 방향 모두 고려

## 주요 특징

### 인덱싱 알고리즘

- **HNSW(Hierarchical Navigable Small World)**: 그래프 기반, 빠른 검색 속도
- **IVF(Inverted File Index)**: 클러스터링 기반, 메모리 효율적
- **FAISS**: Facebook이 개발한 고성능 유사도 검색 라이브러리

### 전통적 DB와의 차이

- 관계형 DB: `WHERE age = 30` (정확한 매칭)
- Vector DB: "30살과 비슷한 나이대" (근사 검색)
- NoSQL: 스키마 유연성 제공
- Vector DB: 의미적 유사성 기반 검색

## 주요 사용 사례

### AI/ML 애플리케이션

- **RAG(Retrieval-Augmented Generation)**: LLM에 관련 문서 제공
- **의미론적 검색**: 키워드가 아닌 의미로 검색
- **추천 시스템**: 사용자 선호와 유사한 항목 추천
- **이미지 검색**: 시각적으로 유사한 이미지 찾기

### 실전 예시

- 고객 지원 챗봇: 과거 유사 질문의 답변 검색
- 음악 스트리밍: 취향에 맞는 곡 추천
- 얼굴 인식: 데이터베이스에서 유사 얼굴 매칭

## 대표적인 Vector DB

### 오픈소스

- **Milvus**: 클라우드 네이티브, 확장성 높음
- **Weaviate**: GraphQL 지원, 하이브리드 검색
- **Qdrant**: Rust 기반, 고성능

### 상용 서비스

- **Pinecone**: 관리형 서비스, 간편한 API
- **Chroma**: 임베딩 함수 내장, 개발자 친화적
- **Supabase Vector**: PostgreSQL 확장, pgvector 기반

## 전체 요약

- Vector DB는 고차원 벡터 데이터의 유사도 검색에 특화된 데이터베이스
- 벡터 임베딩을 통해 텍스트, 이미지 등을 수치화하여 저장
- 코사인 유사도, 유클리드 거리 등으로 벡터 간 유사성 측정
- HNSW, IVF 같은 특수 인덱싱으로 대규모 데이터에서 빠른 검색 제공
- AI 챗봇, 추천 시스템, 의미론적 검색 등에서 핵심 기술로 활용
- 전통적 DB의 정확한 매칭과 달리 "가장 유사한" 결과 반환
- Milvus, Pinecone, Weaviate 등 다양한 오픈소스/상용 솔루션 존재
