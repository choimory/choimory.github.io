# 개요

Pulumi는 Infrastructure as Code(IaC) 도구로, Terraform처럼 클라우드 인프라를 코드로 정의하고 관리하는 플랫폼이다. 가장 큰 특징은 HCL 같은 전용 DSL 대신 범용 프로그래밍 언어(TypeScript, Python, Go, C# 등)를 그대로 사용한다는 점이다.

# Pulumi vs Terraform

| 항목 | Pulumi | Terraform |
| --- | --- | --- |
| 언어 | TypeScript, Python, Go, C# 등 | HCL (전용 DSL) |
| 상태 저장 | Pulumi Cloud / Self-hosted | Local / S3 등 Backend |
| 로직 표현 | 조건문/반복문 등 완전한 프로그래밍 가능 | 제한적 (count, for_each) |
| 학습 곡선 | 프로그래머에게 낮음 | DSL 별도 학습 필요 |
| Provider 수 | 비교적 적음 (Terraform bridge 존재) | 매우 많음 |
| 커뮤니티 | 성장 중 | 매우 성숙 |

# 핵심 개념

- **Project** : Pulumi 프로그램의 단위. `Pulumi.yaml`이 루트에 위치하며 언어/런타임을 정의
- **Stack** : 같은 Project를 여러 환경(dev, staging, prod)에 독립 배포하는 단위. Terraform workspace와 유사
- **State** : 인프라 현재 상태 저장. Terraform `.tfstate`에 해당. 기본은 Pulumi Cloud, S3 등 self-hosted도 지원
- **Resource** : 클라우드 리소스를 표현하는 객체

# 코드 예시 (TypeScript)

```tsx
import * as pulumi from "@pulumi/pulumi";
import * as aws from "@pulumi/aws";

const bucket = new aws.s3.Bucket("my-bucket", {
    acl: "private",
    tags: {
        Environment: pulumi.getStack(),
    },
});

// 반복문으로 여러 리소스 생성
const buckets = ["logs", "assets", "backups"].map(name =>
    new aws.s3.Bucket(`bucket-${name}`, { acl: "private" })
);
```

# CLI 주요 명령어

```bash
pulumi new aws-typescript   # 프로젝트 초기화
pulumi stack init dev        # 스택 생성
pulumi preview               # 변경사항 미리보기 (terraform plan)
pulumi up                    # 배포 실행 (terraform apply)
pulumi destroy               # 리소스 제거
pulumi stack                 # 스택 상태 조회
```

# 장단점

## **장점**

- 기존 프로그래밍 언어 사용 → IDE 자동완성, 타입 체크, 테스트 프레임워크 활용 가능
- 복잡한 로직(동적 리소스 생성, 함수 추상화)을 자연스럽게 표현
- Terraform Provider를 bridge로 사용 가능

## **단점**

- Terraform 대비 커뮤니티/레퍼런스 부족
- 기본 상태 저장소가 Pulumi Cloud(SaaS) → self-hosted 시 별도 설정 필요
- 팀 내 언어 표준화 필요

# NCP와의 관계

NCP 전용 Pulumi Provider는 공식 제공되지 않음. Terraform Provider가 공식 지원되므로 NCP 인프라 관리에는 Terraform이 더 안정적인 선택이다. Terraform bridge를 통한 사용은 이론적으로 가능하나 검증된 레퍼런스가 적음.
