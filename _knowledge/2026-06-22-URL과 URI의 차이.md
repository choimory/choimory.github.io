# URL과 URI의 차이

## 개요

URL과 URI는 인터넷에서 리소스를 식별하고 위치를 나타내는 데 사용되는 핵심 개념입니다. 두 용어가 혼용되어 사용되는 경우가 많지만, 정확히는 URI가 더 넓은 개념이고 URL은 그 하위 개념에 속합니다.

---

## URI란 무엇인가

### 정의

URI(Uniform Resource Identifier)는 "통합 자원 식별자"를 의미합니다. 인터넷 상의 자원을 식별하기 위한 문자열의 표준 형식을 정의한 개념입니다.

#### 핵심 의미

URI의 핵심은 "식별"입니다. 어떤 자원이 "무엇인지"를 나타내는 데 초점이 있으며, 그 자원을 어떻게 찾아갈 수 있는지는 URI의 필수 요소가 아닙니다.

### URI의 하위 분류

URI는 크게 두 가지 방식으로 자원을 식별합니다.

#### URL (Uniform Resource Locator)

자원의 "위치"를 기반으로 식별하는 방식입니다. 어디에 있는지를 알려주기 때문에, 해당 위치로 찾아가면 자원을 얻을 수 있습니다.

#### URN (Uniform Resource Name)

자원의 "이름"을 기반으로 식별하는 방식입니다. 위치와 무관하게 고유한 이름 자체로 자원을 가리킵니다. 예를 들어 ISBN 번호로 책을 식별하는 것이 URN의 대표적인 사례입니다.

```
urn:isbn:0451450523
```

이 경우 책이 실제로 어디에 있는지는 알려주지 않지만, 그 책이 "무엇인지"는 고유하게 식별합니다.

---

## URL이란 무엇인가

### 정의

URL(Uniform Resource Locator)은 "통합 자원 위치"를 의미합니다. 자원이 네트워크 상에서 어디에 위치하는지, 그리고 그 자원에 어떻게 접근(프로토콜)할 수 있는지를 함께 나타냅니다.

### URL의 구성 요소

일반적인 URL은 다음과 같은 구조로 이루어집니다.

```
https://www.example.com:443/path/to/page?query=value#section
```

#### scheme (프로토콜)

`https`처럼 어떤 프로토콜로 접근할지를 나타냅니다.

#### host (호스트)

`www.example.com`처럼 서버의 도메인이나 IP 주소를 나타냅니다.

#### port (포트)

`443`처럼 접속할 포트 번호입니다. 생략 시 프로토콜의 기본 포트가 사용됩니다.

#### path (경로)

`/path/to/page`처럼 서버 내에서 자원의 위치를 나타냅니다.

#### query (쿼리)

`?query=value`처럼 서버에 전달하는 추가 매개변수입니다.

#### fragment (프래그먼트)

`#section`처럼 문서 내 특정 부분을 가리키는 식별자입니다.

---

## URI와 URL의 관계

### 포함 관계

URI는 URL과 URN을 포괄하는 상위 집합입니다. 모든 URL은 URI이지만, 모든 URI가 URL인 것은 아닙니다.

```
URI
├── URL (위치로 식별, scheme + host + path 등 포함)
└── URN (이름으로 식별, 위치 정보 불필요)
```

### 현실에서의 사용

다만 실무에서는 URN이 거의 쓰이지 않고 거의 모든 식별이 URL 형태로 이루어지기 때문에, "URI"와 "URL"이라는 용어가 사실상 같은 의미로 혼용되는 경우가 많습니다. RFC 3986 명세에서도 URL이라는 용어보다는 URI라는 용어를 표준 용어로 사용할 것을 권장하고 있습니다.

#### 비교 표

| 구분 | URI | URL |
| --- | --- | --- |
| 의미 | 통합 자원 식별자 | 통합 자원 위치 |
| 역할 | 자원을 식별 | 자원의 위치를 식별 |
| 위치 정보 필수 여부 | X | O |
| URN 포함 여부 | O | X |
| 예시 | urn:isbn:0451450523, https://example.com | https://example.com/page |

---

## 전체 요약

URI는 인터넷 자원을 식별하는 가장 넓은 개념이며, 그 하위에 위치 기반으로 식별하는 URL과 이름 기반으로 식별하는 URN이 포함됩니다. URL은 scheme, host, path 등 자원에 실제로 접근할 수 있는 경로 정보를 포함하는 반면, URN은 위치와 무관하게 고유한 이름만으로 자원을 식별합니다. 실무에서는 거의 모든 식별자가 URL 형태이기 때문에 URI와 URL이 혼용되어 쓰이는 경우가 많지만, 개념적으로는 URI가 더 상위 개념이라는 점을 이해하는 것이 중요합니다.
