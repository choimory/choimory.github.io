# 기본

- 한국어로 대화한다.
- 작업을 바로 시작하지 말고 일단 작업 계획을 설명해주고, 작업 끝난 후에도 문서를 바로 작성하지 말고 일단 작업 내용을 설명해줄 것.
- 작업 진행, 문서 작성, 테스트 진행은 물어보고 허락을 받아야 한다. 내 지시 전까지는 기본적으로 대화로 의견을 주고 받는다.
- 테스트는 직접 진행하지 말고 방법을 알려주기.
- 작업 마무리하자 하면, 작업 전체 요약을 done1~.md에 적고, CALENDAR.md에 작업사항을 추가하고 README.md에 변경사항을 반영하는 것으로 마무리한다.
- 프로젝트 전체 정보는 README.md에 작성하고, 변경사항은 반영하며, 분석시에는 참고한다.
- 특정 작업과 관련된 상세한 히스토리를 참고할땐, .agents/CALENDAR.md를 확인하여 해당 작업의 디렉토리 내 문서들을 참고한다.
- 민감한 정보는 마스킹 한다. (api-secret-key, ssh-private-key, token-secret-key, terraform-resource-id...)

# 문서 디렉토리

## 문서 디렉토리 규칙 

- 내가 직접 문서를 만들어 놓을 때도 있지만, 그렇지 않을 때는 문서 디렉토리 양식대로 문서를 생성하며 작업을 진행한다.
- 해당 디렉토리 안에서 작업 개요 및 분석과 계획은 plan1~.md, 작업 내용은 task1~.md, 작업 완료 요약은 done1~.md에 작성한다.
- 문서 마크다운 파일은 utf-8로 작성한다.
- 자세한 사항은 하단 문서 디렉토리 양식란 참고.
- 디렉토리명은 영문으로 작성한다

## 문서 디렉토리 양식

```markdown
# 문서 디렉토리 양식 (.agents/CALENDAR.md, .agents/year/month/day/작업내용/plan1~.md, task1~.md, done1~.md)

.agents/CALENDAR.md
.agents/year/month/day/작업내용/plan1.md
                               plan2.md
                               task1.md
                               task2.md
                               task3.md
                               done1.md
                               done2.md

---

# 예시 (.agents/CALENDAR.md, .agents/2025/01/01/ssl-apply/plan1.md, plan2.md, task1.md, task2.md, task3.md, done1.md)

.agents/CALENDAR.md
.agents/2025/01/01/ssl-apply/plan1.md
                             plan2.md
                             task1.md
                             task2.md
                             task3.md
                             done1.md
                             done2.md
```

# 문서 작성

## 문서 작성 규칙

- 기본적으로는 대화로 이야기하며, 문서 작성은 내가 작성 요청할 때만 작성한다.
- 문서 작성 요청할 땐 기본적으로 기존 글들을 수정하지 않는다.
- 질문과 대화를 1:1 형식으로 `---`로 구분선을 그어 구분해가며, 밑에 새로 추가하는 형식으로 적을 것.
- 내 질문 내용을 적을땐 요약하거나 변경하지 말고 그대로 적을것.
- 문서는 H1 > H2 > H3 형식으로 주제를 구분하며 적을 것.
- 문서 맨 위에 목차도 적고, 대화 내용을 추가할 때마다 업데이트 할 것.
- 문서 마크다운 파일은 utf-8로 작성한다.
- 자세한 사항은 하단 문서 작성 양식란 참고.

## 문서 작성 양식


# CALENDAR

## CALENDAR.md 작성 규칙 

- 작업이 마무리되면 작업 일자별 디렉토리들의 작업 사항을 CALENDAR.md에 추가하여 마무리 한다.
- 문서 디렉토리의 일자별 작업 내용을 하나의 문서에 한 줄로 정리하고자 함.
- 자세한 사항은 하단 CALENDAR.md 양식란 참고.

## CALENDAR.md 양식

```markdown
# 2025

- yyyy-MM-dd: 간략한 작업주제(디렉토리명)
- 2025-01-02: ssl 적용작업(ssl-apply)
- 2025-01-03: 회원 api 리팩토링 작업(member-api-refactor), 프론트 마이페이지 적용 작업(front-mypage-apply)

# 2026
...
  
```
